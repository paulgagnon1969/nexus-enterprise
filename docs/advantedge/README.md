> **The definitive collection of competitive advantages built into NCC.**

## Document Taxonomy

| Type | Purpose | Structure |
|------|---------|-----------|
| **Book** | Sequential narrative | Chapters with flow, read start to finish |
| **Manual** | Functional reference | Contextual how-to, look up what you need |
| **advantEDGE Brief** | Competitive showcase | What → Why → How it's different |
| **eDoc** | Standalone document | Atomic building block for all of the above |

## Brief Structure

Every advantEDGE brief follows this format:

```
# NCC advantEDGE™: [Feature Name]
## Tagline            — One sentence, punchy
## What It Is         — Technical description (2-3 sentences)
## Why It Matters     — The business problem it solves
## How It's Different — What competitors don't have
## Key Capabilities   — Bullet list of specifics
## Sales Talking Points — Ready-to-use phrases for demos/pitches
## Technical Notes    — Implementation details for internal team
```

## Usage

- **Sales**: Pull the relevant brief before a demo or proposal. Use the talking points verbatim.
- **Marketing**: Extract taglines and differentiators for website/collateral.
- **Product**: Reference when prioritizing features — if it's not in advantEDGE, should it be?
- **Onboarding**: New team members read the index to understand what makes NCC special.
